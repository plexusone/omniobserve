## [v0.6.0] - 2026-03-01

### Highlights

- Organization rename from agentplexus to plexusone
- Built-in slog provider for local trace logging during development and debugging
- Structured evaluation integration for connecting llmops with sevaluation workflows

### Added

- `llmops/slog` provider for local structured logging of trace events ([`9b7dc4e`](https://github.com/plexusone/omniobserve/commit/9b7dc4e))
- `WithLogger` option in `ClientOptions` for custom slog.Logger configuration ([`9b7dc4e`](https://github.com/plexusone/omniobserve/commit/9b7dc4e))
- `integrations/sevaluation` package for structured-evaluation integration ([`a7816f6`](https://github.com/plexusone/omniobserve/commit/a7816f6))

### Changed

- **Breaking**: Module path changed from `github.com/agentplexus/omniobserve` to `github.com/plexusone/omniobserve`

### Fixed

- gosec warnings for APIKey struct fields (G117) and HTTP client SSRF (G704) ([`7914c6a`](https://github.com/plexusone/omniobserve/commit/7914c6a))

### Dependencies

- Upgraded `github.com/lib/pq` from v1.10.9 to v1.11.1 ([`3a1ccde`](https://github.com/plexusone/omniobserve/commit/3a1ccde))
- Upgraded `github.com/plexusone/omnillm` from v0.11.1 to v0.13.0
- Upgraded `github.com/plexusone/structured-evaluation` from v0.2.0 to v0.3.0

### Documentation

- Observability integration guide for stats-agent-team ([`4cdffaa`](https://github.com/plexusone/omniobserve/commit/4cdffaa))
- README updated with slog provider in supported providers table
- Updated all documentation to use new module path

## [v0.5.0] - 2026-01-03

### Highlights

- Comprehensive evaluation metrics for assessing LLM outputs with both code-based and LLM-based metrics
- Streamlined provider architecture with Opik and Phoenix adapters moved to standalone SDKs

### Added

- `AnnotationManager` interface for span/trace annotations with `CreateAnnotation` and `ListAnnotations` methods ([`2baa46f`](https://github.com/plexusone/omniobserve/commit/2baa46f))
- `DatasetManager` methods: `GetDatasetByID` and `DeleteDataset` ([`2baa46f`](https://github.com/plexusone/omniobserve/commit/2baa46f))
- Prompt model/provider options: `WithPromptModel`, `WithPromptProvider`, `ModelName`, `ModelProvider` fields ([`7cea000`](https://github.com/plexusone/omniobserve/commit/7cea000))
- OmniLLM hook auto-creates traces when none exists in context ([`ef67b89`](https://github.com/plexusone/omniobserve/commit/ef67b89))
- Trace context helpers: `contextWithTrace` and `traceFromContext` ([`7cea000`](https://github.com/plexusone/omniobserve/commit/7cea000))
- `llmops/metrics` package with LLM-based metrics: `HallucinationMetric`, `RelevanceMetric`, `QACorrectnessMetric`, `ToxicityMetric` ([`079fe35`](https://github.com/plexusone/omniobserve/commit/079fe35))
- `llmops/metrics` package with code-based metrics: `ExactMatchMetric`, `RegexMetric`, `ContainsMetric` ([`079fe35`](https://github.com/plexusone/omniobserve/commit/079fe35))
- `examples/evaluation` demonstrating metrics usage ([`f582c2a`](https://github.com/plexusone/omniobserve/commit/f582c2a))

### Changed

- **BREAKING:** Provider adapters moved to standalone SDKs: Opik to `github.com/plexusone/go-opik/llmops`, Phoenix to `github.com/plexusone/go-phoenix/llmops` ([`333f6fd`](https://github.com/plexusone/omniobserve/commit/333f6fd))

### Removed

- `llmops/opik` adapter (moved to go-opik) ([`333f6fd`](https://github.com/plexusone/omniobserve/commit/333f6fd))
- `llmops/phoenix` adapter (moved to go-phoenix) ([`333f6fd`](https://github.com/plexusone/omniobserve/commit/333f6fd))
- `sdk/phoenix` package (use go-phoenix directly) ([`333f6fd`](https://github.com/plexusone/omniobserve/commit/333f6fd))

### Fixed

- Redundant newlines in example Println statements ([`6758f68`](https://github.com/plexusone/omniobserve/commit/6758f68))

### Documentation
